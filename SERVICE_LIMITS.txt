==========================================================
ADC — SERVICE LIMITS & CAPACITY (quick reference)
Estimates from architecture review, not load-tested.
==========================================================

[1] VERCEL  (frontend — Next.js)
  Plan: Hobby (non-commercial ToS — move to Pro for a real store).
  Auto-scales; serving pages is NOT the bottleneck.
  Limits: bandwidth + function invocations capped on Hobby.
  Capacity: thousands of concurrent page loads — fine.
  Action: upgrade Hobby -> Pro before launch.

[2] RAILWAY  (backend — Node/Express)
  Setup: SINGLE instance, one CPU thread (no replicas).
  Light JSON is fast. With pool=30, THIS is now the ceiling.
  Limits: ~100-200 req/sec sustained as configured.
  Capacity: ~500-800 active users comfortable, ~1500 burst.
  Action: bigger instance or add replicas (stateless = easy).

[3] SUPABASE  (Postgres DB + Auth)
  DB pool: max 30 connections (db.js). Safe — free tier allows
           ~60 direct, so 30 = 2x headroom for one instance.
  Key: 30 conns != 30 users. Each query frees its connection
       in ~10ms, so 30 serves HUNDREDS of users.
  Pooler: Supavisor session mode (5432). For more, use the
          transaction pooler (6543) and raise max further.
  Auth: logins/sessions effectively unlimited (JWT).
  Capacity: no longer the tight limit (Railway is).

[4] DELHIVERY  (shipping)
  Auth: API token (Authorization: Token ...). Working.
  All APIs validated; shipment+label BLOCKED on wallet balance.
  No rate issue at our volume; add prepaid balance to ship.
  Label PDF (with logo) pulled via API into admin.
  Action: keep Delhivery wallet funded.

[5] RAZORPAY  (payments)
  Checkout currently SIMULATED (no real charge wired yet).
  When live: Razorpay handles its own scaling, not a limit.
  Verify hook (/payment/verify) is where it plugs in.
  Action: wire real Razorpay keys + signature verification.

----------------------------------------------------------
OVERALL (with DB pool = 30)
  Logged-in & idle ...... unlimited (thousands+)
  Active shoppers ....... ~500-800 comfortable, ~1500 burst
  Requests/sec .......... ~100-200 sustained (Railway, 1 instance)
  Simultaneous checkouts  ~40-60
  Ceiling is now Railway's single instance, NOT the DB.
  Next fix: bigger Railway instance / replicas -> ~2000-3000+.
